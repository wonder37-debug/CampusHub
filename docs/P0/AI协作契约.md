# **CampusHub 项目 AI 协作契约 (Phase 0\)**

**团队名称：** CampusHub 智协小组

**生效日期：** 2026年4月11日

## **第一条：AI 使用基本原则**

1. **工程师主体责任：** AI 仅作为辅助增强工具，所有交付的代码、文档及架构设计的最终责任人均为相关环节的负责人（人）。  
2. **审查后合入：** 严禁任何 AI 生成的内容未经人工逐行审查、测试及逻辑核对便直接并入主分支。  
3. **防止技术退化：** 团队成员需有意识地维持手写代码能力，确保在无 AI 环境下仍具备独立开发核心业务逻辑的能力。

## **第二条：AI 使用场景与边界规范**

### **2.1 允许/推荐使用场景**

* **辅助编码：** 编写常规 README 文档、生成 CRUD 代码模板、编写基础 POJO/DTO 类、实现通用的算法逻辑。  
* **质量保障：** 根据现有代码自动生成 JUnit 单元测试模板、辅助定位复杂的 Debug 堆栈信息。  
* **文档增强：** 辅助润色 SRS 需求文档、生成 API 接口的初始 Swagger 注释。

### **2.2 受限使用场景（需附带深度审查说明）**

* **核心业务：** 复杂的数据库事务逻辑处理、涉及金钱或订单状态流转的敏感代码。  
* **安全模块：** 核心权限验证（Spring Security）、JWT 生成与校验逻辑。  
* **架构决策：** ADR（架构决策记录）的初稿可参考 AI 建议，但最终选型必须基于团队调研。

### **2.3 严禁使用场景**

* **一键提交：** 严禁在 Cursor 等工具中通过 "Apply All" 或 "Composer" 模式生成大段代码后，未做任何修改与测试即直接 Commit。  
* **避重就轻：** 严禁使用 AI 逃避对核心算法和底层原理的学习与理解。

## **第三条：过程记录与工程规范**

1. **日志留痕：** \- 团队需保留所有关键任务（如 P2 架构设计、P4 核心编码）的 Prompt 交互日志。  
   * 交互记录统一存放在 docs/AI\_Logs/ 目录下。  
2. **Git 提交标注：** \- 所有的 Git Commit Message 必须遵循以下标签规范：  
   * \[AI-assisted\] description: 包含 AI 生成或辅助优化的内容。  
   * \[Human-written\] description: 完全由人工编写的核心逻辑。  
3. **反思日志：** 每个阶段结束后，全员需共同填写《AI 协作反思日志》，复盘 AI 带来的提效值与潜在风险。

## **第四条：代码合入与质量门禁**

1. **双人审查（Code Review）：** \- 所有标注为 \[AI-assisted\] 的代码段，必须通过至少一名非编写者成员的交叉审查。  
   * 利用 **CodeRabbit** 进行初步自动化审查，人工复核其修改建议。  
2. **手写指标约束：** \- **底线要求：** 每位成员在每个编码阶段（特别是 P4-P5），必须有至少 **2 个核心业务函数** 完全由手写完成，并在 Review 时进行重点展示。

## **第五条：违约处理机制**

1. **代码讲解：** 若发现未经审查合入 AI 生成的代码，责任人需在下周周会上对该模块进行“逻辑深度拆解”，证明其完全掌握了该代码段的运行机制。  
2. **强制重构：** 对于逻辑混乱、存在严重安全隐患或“幻觉”产生的 AI 代码，责任人必须在 24 小时内进行手动重构并重新发起 PR。
