# Bug 修复日志

**说明：** 本日志按日期顺序记录 dev 分支中的主要修复与联调工作。每一天都用同样的四项格式记录：问题现象、根因分析、修复方案、验证结果。

---

## 2026-05-11
- **问题现象：** 初步搭建展示层框架，前端页面骨架和基础路由尚未完全串联。
- **根因分析：** 项目处于展示层起步阶段，页面结构和路由还没有统一收拢。
- **修复方案：** 先完成前端基础结构整理，为后续页面联调和样式调整预留入口。
- **验证结果：** 展示层框架完成，页面可以作为后续开发的统一起点。

## 2026-05-12
- **问题现象：** 页面文案不够清晰，非主页的 demo-strip 组件对主流程有干扰。
- **根因分析：** 页面展示信息偏多，信息层级不够明确。
- **修复方案：** 优化文案并隐藏非主页 demo-strip，减少视觉干扰。
- **验证结果：** 页面表达更清楚，主流程信息更集中。

## 2026-05-14
- **问题现象：** 页面结构分散，首页、需求页、订单页和个人页之间缺少统一串联。
- **根因分析：** 前端展示层仍以分散组件和预设数据为主，缺少统一的数据流。
- **修复方案：** 重新设计页面结构，并同步更新 store、类型和多个视图。
- **验证结果：** 页面展示链路开始成型，为切换后端接口数据做好准备。

## 2026-05-15
- **问题现象：** 前后端开始联调时，部分接口、认证和数据来源还没有完全对齐。
- **根因分析：** 后端邮箱逻辑、token 解析和前端代理配置都处于第一次对接阶段。
- **修复方案：** 补齐邮箱逻辑和前端 Vite 代理，修复 token 解析问题，并把前端预设数据切换为后端接口。
- **验证结果：** 认证、需求、订单、评价、通知等控制器与基础逻辑完成第一轮对接。

## 2026-05-16
- **问题现象：** 本地数据库初始化和用户持久化还不完整。
- **根因分析：** 本地 MySQL 表结构、初始化脚本和用户仓库实现尚未统一。
- **修复方案：** 初始化本地 MySQL 表结构，补充建表脚本和使用说明，同时完成 MyBatis 版 UserRepository。
- **验证结果：** 用户数据可以落到数据库，认证逻辑开始具备可运行的持久化基础。

## 2026-05-17
- **问题现象：** 需求、订单、评价和通知的持久化能力都还不够完整，状态流转容易出问题。
- **根因分析：** 各模块 DAO 层和表结构还没有完全对齐，部分模块缺少并发防重和级联保存。
- **修复方案：** 完成 DemandRepository、订单级联持久化、评价并发防重和通知 DAO 与测试。
- **验证结果：** 各模块的数据落库与查询能力明显完善，核心业务流转更稳定。

## 2026-05-18
- **问题现象：** 后台统计、用户管理和详情页展示与数据库字段不一致。
- **根因分析：** is_approved、updated_at 等字段在数据层和展示层没有完全统一。
- **修复方案：** 扩展后台查询能力，补齐相关字段并统一实体、DTO、仓库和测试资源的字段定义。
- **验证结果：** 后台展示和详情页时间字段表现更一致，字段对齐问题基本收敛。

## 2026-05-19
- **问题现象：** 服务器重启后数据容易丢失，系统有时仍然优先使用内存仓库。
- **根因分析：** 仓库切换策略不正确，数据库实现存在时没有被优先选中。
- **修复方案：** 调整仓库优先级，让存在数据库实现时优先使用数据库。
- **验证结果：** 数据可以稳定保存在数据库中，重启丢数据的问题被解决。

## 2026-05-22
- **问题现象：** 本地初始化脚本在重复执行时会出现创表冲突。
- **根因分析：** 初始化脚本没有处理重复建表的场景。
- **修复方案：** 修复脚本中的重复创表问题，让脚本可重复执行。
- **验证结果：** 本地初始化流程更稳定，可以反复执行。

## 2026-05-24
- **问题现象：** 注册时邮箱填写容易出错，用户常把完整邮箱直接填进前缀框。
- **根因分析：** 注册表单缺少邮箱前缀与后缀的显式约束。
- **修复方案：** 修改注册流程，新增邮箱前缀输入检测和后缀选择功能。
- **验证结果：** 注册邮箱填写更清晰，误填概率下降。

## 2026-05-25
- **问题现象：** 登录态会因刷新丢失，通知批量已读不够稳定，注册验证码和错误提示也不够完善，CI/CD 与集成测试还没有完全打通。
- **根因分析：** 登录信息只存在运行时状态；通知批处理缺少失败隔离；邮箱发送与校验逻辑未形成统一约束；流水线和测试覆盖还不完整。
- **修复方案：** 将登录态持久化到 localStorage，优化通知已读为异步操作，完善邮箱发送与校验逻辑，新增头像上传与验证码状态提示，并搭建 CI/CD 与集成测试。
- **验证结果：** 页面刷新后可保持登录态，通知批量操作更稳定，注册流程和流水线验证能力也补齐了。
