## 4. 本轮的临时统一方案

P1 与 P3 在“分类体系”上并不完全一致：

- P1 偏业务场景分类，如 跑腿 / 学习 / 生活 / 二手
- P3 API 已固定为更具体的接口枚举，如 `EXPRESS`、`STUDY_TUTORING`、`SECOND_HAND`、`TEAM_UP`、`OTHER`

本轮取舍：

- 代码层优先沿用 P3 API 规范的分类枚举
- 同时把 P1 中后续一定会影响数据结构的 `campusZone`、`anonymous` 先纳入模型

这样可以减少后续 Controller / OpenAPI 联调返工。

## 本轮已落地业务规则

本轮已经写入代码并通过测试的规则包括：

- 标题长度校验
- 描述最大长度校验
- 校区必填
- 分类必填
- `reward >= 0`
- `endTime` 不能早于 `startTime`
- 标签数量不超过 20
- 敏感词命中时阻止发布
- 发布成功默认状态为 `PENDING`
- 匿名需求详情隐藏真实 `publisherId`
- 非发布者不能修改需求
- 已完成需求不能再修改
