# **“校邻互助”校园服务平台 \- 调研方案**

**版本：** 1.2

**日期：** 2024-03-24

**调研问卷设计**

### **第一部分：功能板块与核心需求**

**1\. 您希望平台包含哪些功能或板块？请列举您认为在校园生活中最需要的互助场景。**

**2\. 在您上述提到的功能或板块中，您认为最核心、最不可或缺的一个是？**

**3\. 在日常生活中，您还有哪些需要他人参与协助的特殊需求（目前难以通过现有社交软件解决的）？**

*(例如：跨校区带书、寝室紧急借物、深夜紧急求助等)*

### **第二部分：身份核验与平台治理**

**4\. 您认为通过“学号认证”作为唯一的注册核验方式是否合适？**

* A. 合适，能确保校内人员纯净，安全性高  
* B. 不合适，希望能有更多元或更隐私的核验方式（如学生邮箱等）

**5\. 为了让平台维持良好的氛围并长久运行，您认为平台应该重点管理或限制哪些方面的内容/行为？**

### **第三部分：商业模式与参与意愿**

**6\. 您认为平台应该通过何种方式获得运营资金或盈利以维持服务器开销？**

**7\. 如果平台招募学生志愿管理员，您的应聘意愿是？**

*(分值 1-10，1 表示无意愿，10 表示极度渴望)*

**8\. 要是平台正式投入使用了，您的真实使用意愿是？**

*(分值 1-10，1 表示绝不使用，10 表示高频使用)*

**9\. 您对本平台的建设（如界面风格、功能逻辑等）还有哪些具体的建议？**

### **第四部分：互助逻辑与闭环**

**10\. 您希望在互助完成后获得怎样的酬劳？您对使用“虚拟积分”代替“现金”的看法是？**

**11\. 在互助的过程中，您认为是否有必要在平台内内置实时聊天功能？**

* A. 有必要（方便即时沟通位置、时间等细节）  
* B. 没必要（更倾向于直接通过微信或电话联系）

**12\. 您认为平台应该通过什么标准或流程来判定一个互助任务已经“成功完成”？**

# **3\. 访谈提纲**

1. **痛点溯源：** 请分享一次你最急需帮助却求助无门的经历，当时你是如何尝试解决的？  
2. **信任机制：** 在陌生人互助中，对方展示哪些信息（如：实名认证、历史评价、专业年级）会让你产生信任感？  
3. **纠纷处理：** 如果对方接单后爽约或完成质量极差，你认为平台最合理的惩罚机制是什么？  
4. **隐私平衡：** 在发布某些隐私需求（如代买药品、心理求助）时，你对“匿名发布”的需求程度如何？
