# 数据库配置快速指南（团队成员必读）

## 🎯 一句话总结
**为了避免代码库密码泄露与环境冲突，每个团队成员如果需要在本地搭建自己的 MySQL 数据库实例，请使用被 Git 忽略的本地配置文件存放密码。**

---

## 📝 操作步骤

### 1. 确认 MySQL 环境
- 请确保你本地已经安装了 MySQL 8.0 或更高版本。
- 确认 MySQL 服务正在运行（默认端口：3306）。
- 记住你本地 MySQL 的 `root` 密码。

### 2. 创建专属数据库
打开任意 MySQL 客户端（如 DataGrip、Navicat、DBeaver 或命令行），执行以下语句创建支持表情符号的纯净数据库：

```sql
CREATE DATABASE IF NOT EXISTS campushub 
DEFAULT CHARACTER SET utf8mb4 
COLLATE utf8mb4_unicode_ci;

USE campushub;
```

### 3. 导入最新表结构
数据库建好后，需要导入所有表结构。
- **文件位置**：请在项目目录中找到最新的建表脚本（backend\src\main\resources\init_schema.sql）。
- **执行**：在 MySQL 客户端中执行该脚本的全部内容。执行完毕后刷新，应该能看到 `sys_user`、`ord_demand` 等 8 张表。

### 4. 配置本地真实密码（核心步骤）
为了安全，主配置文件 `application.properties` 中使用的是占位符，**不要去改它**。你需要通过以下两种方式之一来配置你的本地密码：

**▶ 方式 A：创建本地专用配置文件（推荐）**
1. 进入目录 `backend/src/main/resources/`

3. 新建文件 `application-local.properties`，填入你的 MySQL 密码：
   ```properties
   spring.datasource.password=你的真实MySQL密码
   ```
   *(注：该文件已被 `.gitignore` 拦截，不会被推送到 GitLab)*

**▶ 方式 B：设置 IDE 环境变量**
如果不建文件，可以在 IDEA 运行配置（Run/Debug Configurations -> Environment Variables）中添加：
`DB_PASSWORD=你的真实MySQL密码`

### 5. 启动测试

在 IDE 中找到 `BackendApplication.java` (backend/src/main/java/com/campushub/backend/BackendApplication.java) 并运行，或者在终端执行：
```bash
cd backend
./mvnw spring-boot:run
```
如果控制台没有打印红色的 DataSource 报错，且成功启动 Tomcat，则本地数据库跑通。
