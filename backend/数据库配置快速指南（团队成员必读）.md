# 数据库配置快速指南（团队成员必读）

## 🎯 一句话总结
**启动后端时 Spring Boot 会自动建表，你只需确保 MySQL 在运行并配好自己的密码即可。**

---

## 📋 环境前置条件

| 工具 | 版本要求 | 说明 |
|---|---|---|
| JDK | 21 | 后端编译运行 |
| MySQL | 8.x | 默认连接 `localhost:3306`，数据库名 `campushub` |
| Node.js | 18+ | 前端构建运行 |
| Maven | 无需安装 | 项目内置 `mvnw`，直接使用即可 |

---

## 📝 操作步骤

### 1. 确认 MySQL 环境
- 确保本地已安装 MySQL 8.0 或更高版本。
- 确认 MySQL 服务正在运行（默认端口：3306）。
- 记住你本地 MySQL 的 `root` 密码。

### 2. 创建数据库（可选）

打开任意 MySQL 客户端（如 DataGrip、Navicat、DBeaver 或命令行），执行：

```sql
CREATE DATABASE IF NOT EXISTS campushub
DEFAULT CHARACTER SET utf8mb4
COLLATE utf8mb4_unicode_ci;
```

> 💡 这一步是**可选的**。因为连接 URL 中已包含 `createDatabaseIfNotExist=true` 参数，Spring Boot 启动时会自动创建数据库。但手动创建可以确保使用 `utf8mb4` 字符集。

### 3. 配置本地密码（核心步骤）

主配置文件 `application.properties` 中密码使用了占位符 `${DB_PASSWORD:root}`，**不要去改它**。请通过以下任一方式配置你的本地密码：

**▶ 方式 A：本地配置文件（推荐）**

在 `backend/src/main/resources/` 下新建 `application-local.properties`：

```properties
spring.datasource.password=你的真实MySQL密码
```

> ✅ 该文件已被 `.gitignore` 拦截，不会被推送到 GitLab，可以放心存放密码。

**▶ 方式 B：IDE 环境变量**

在 IDEA 运行配置（Run/Debug Configurations → Environment Variables）中添加：

```
DB_PASSWORD=你的真实MySQL密码
```

### 4. 启动后端

**表结构无需手动导入！** 项目已配置 `spring.sql.init.mode=always`，Spring Boot 启动时会自动执行 `init_schema.sql` 完成全部 8 张表的创建和默认数据初始化。

启动方式（二选一）：

- **IDE 启动**：在 IDE 中运行 `BackendApplication.java`
  （`backend/src/main/java/com/campushub/backend/BackendApplication.java`）
- **终端启动**：
  ```powershell
  cd backend
  .\mvnw.cmd spring-boot:run
  ```

启动成功后：
- 后端服务运行在 `http://localhost:8080`
- 控制台无红色 DataSource 报错即表示数据库连接正常

### 5. 启动前端

```powershell
cd frontend
npm install    # 首次需要安装依赖
npm run dev
```

前端开发服务器启动后访问 `http://localhost:5173`，所有 `/api` 请求会自动代理到后端 `8080` 端口。

## 🧪 运行测试

后端测试使用 **H2 内存数据库**，完全不需要本地 MySQL，直接运行即可：

```powershell
cd backend

# 运行全部测试
.\mvnw.cmd test

# 运行指定测试类
.\mvnw.cmd test -Dtest=MyBatisUserRepositoryTest
```

> 测试使用的是 `src/test/resources/` 下的 `schema-*.sql` 文件（H2 兼容语法），
> 与主项目的 `init_schema.sql`（MySQL 语法）是**双轨独立**的，互不影响。

## ❓ 常见问题

| 问题 | 原因 | 解决 |
|---|---|---|
| `Access denied for user 'root'` | MySQL 密码不匹配 | 检查 `application-local.properties` 中的密码是否正确 |
| `Port 8080 already in use` | 端口被占用 | 在 `application.properties` 中修改 `server.port` |
| 前端请求 404 / 502 | 后端未启动 | 先确保后端在 8080 端口运行 |
| `Unsupported class file major version 65` | JDK 版本不对 | 运行 `java -version` 确认是 JDK 21 |
