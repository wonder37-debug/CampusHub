# 邮箱验证码发信配置说明

## 作用

后端注册流程已经支持发送邮箱验证码。
当项目配置了 `spring.mail.*` 相关参数后，`POST /api/v1/auth/email-code` 会向目标邮箱发送验证码邮件。
如果没有配置邮件服务，系统会退化为日志输出，不会阻塞应用启动。

---

## 需要配置的文件

推荐优先修改：

- `backend/src/main/resources/application-local.properties`

通用模板位于：

- `backend/src/main/resources/application.properties`

---

## 最小可用配置

下面这组配置是最少需要补齐的：

```properties
APP_MAIL_FROM=your-account@example.edu.cn

SPRING_MAIL_HOST=smtp.example.com
SPRING_MAIL_PORT=587
SPRING_MAIL_USERNAME=your-account@example.edu.cn
SPRING_MAIL_PASSWORD=your-smtp-password-or-auth-code
SPRING_MAIL_PROTOCOL=smtp
SPRING_MAIL_SMTP_AUTH=true
SPRING_MAIL_SMTP_STARTTLS_ENABLE=true
SPRING_MAIL_SMTP_SSL_ENABLE=false
```

说明：

- `APP_MAIL_FROM`：邮件发件人地址，通常与登录邮箱一致
- `SPRING_MAIL_USERNAME`：SMTP 登录账号
- `SPRING_MAIL_PASSWORD`：SMTP 密码或授权码，很多邮箱平台要求使用授权码而不是登录密码
- `SPRING_MAIL_HOST` 和 `SPRING_MAIL_PORT`：由邮箱服务商提供

## 学校邮箱域名白名单

系统会校验注册邮箱是否属于允许的学校域名，配置项如下：

```properties
APP_AUTH_ALLOWED_EMAIL_DOMAINS=edu.cn,example.edu.cn,campus.edu,test.edu.cn
```

如果你们项目只允许特定学校邮箱，建议改成更精确的域名列表，例如：

```properties
APP_AUTH_ALLOWED_EMAIL_DOMAINS=stu.nju.edu.cn,nju.edu.cn
```

多个域名之间使用英文逗号分隔。

---

## 如何验证是否配置成功

1. 启动后端服务。
2. 调用 `POST /api/v1/auth/email-code`。
3. 请求体示例：

```json
{
  "email": "user@example.edu.cn",
  "studentId": "20260001"
}
```

4. 如果配置正确，目标邮箱会收到标题为“CampusHub 注册验证码”的邮件。
5. 如果没有配置 SMTP，控制台会输出验证码日志，但不会真正发信。

## 常见问题

### 1. 返回 “failed to send verification email”

可能原因：

- SMTP 地址或端口错误
- 用户名或密码错误
- 邮箱平台要求授权码，但填了登录密码
- SSL / STARTTLS 开关与服务商要求不匹配

建议优先检查：

- `SPRING_MAIL_HOST`
- `SPRING_MAIL_PORT`
- `SPRING_MAIL_USERNAME`
- `SPRING_MAIL_PASSWORD`
- `SPRING_MAIL_SMTP_STARTTLS_ENABLE`
- `SPRING_MAIL_SMTP_SSL_ENABLE`

### 2. 返回 “email must be a valid campus email”

说明邮箱格式不合法，或者域名不在 `APP_AUTH_ALLOWED_EMAIL_DOMAINS` 白名单内。

### 3. 本地明明启动了，但没有收到邮件

先看控制台日志：

- 如果只有日志输出验证码，没有真实邮件，大概率是没有成功装配 `JavaMailSender`
- 如果抛出了发信异常，通常是 SMTP 参数不正确

---

## 安全建议

- 不要把真实邮箱密码或授权码提交到 Git 仓库。
- 推荐只在本地 `application-local.properties` 或 IDE 环境变量里填写敏感信息。
- 如果是团队共享开发，优先使用授权码，不要直接使用邮箱登录密码。
